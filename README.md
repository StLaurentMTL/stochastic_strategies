# Stochastic Strategies
## A Songwriting Tool inspired by David Bowie, Brian Eno, and Stochastics

In 1976, following a tumultuous year dealing with severe addictions, David Bowie decamped to the city of Berlin in search of musical reinvention with the legendary producer Brian Eno. During their time recording the albums _Low_, _Heroes_, and _Lodger_, Bowie and Eno sought inspiration from a deck of bamboo cards called _Oblique Strategies_. These cards carried mysterious but thought-provoking remarks that the artist would then have to decipher in their music. Since their use by Bowie and Eno, many artists have used the cards in their own albums, including LCD Soundsystem, Phoenix, The B-52s, MGMT, and Bauhaus.

![Bowie, Eno, Fripp — photo by Sky TV](https://github.com/StLaurentMTL/stochastic_strategies/blob/main/photos/david_brian.jpg)

**Stochastic Strategies** is a songwriting tool that generates creative constraints on demand: a chord progression, a time signature, a tempo, and an Oblique-like prompt — all randomized, all combined into a single "creative dilemma" card. Chord progressions are generated by a Markov chain trained on 600 k+ real songs, so the harmony feels grounded even when it surprises you.

## How it works

### Chord generation — Markov chain + temperature scaling

Chord progressions are sampled from a first-order Markov chain over the seven diatonic scale degrees (I–VII). The transition matrix **P** is fitted offline on the chorus sections of songs from the [Chordonomicon dataset](https://huggingface.co/datasets/ailsntua/Chordonomicon) (Kantarelis et al., 2024 — 666 k songs).

> For a full mathematical treatment of the fitting pipeline — chord parsing, key inference, bigram accumulation, Laplace smoothing, and temperature scaling — see [docs/transition_matrix_methodology.md](docs/transition_matrix_methodology.md).

At generation time, a temperature parameter τ reshapes the matrix:

```
P(τ)ᵢⱼ = softmax(log(Pᵢⱼ) / τ)ⱼ
```

- **τ < 1 (Relaxed)** — sharpens the distribution toward the most probable transitions; progressions stay close to familiar voice-leading
- **τ > 1 (Challenge)** — flattens the distribution toward uniform; progressions become more chromatic and harmonically unexpected

The Shannon entropy rate **H** (bits per chord) is printed on every card as a quantitative difficulty signal. Maximum possible entropy is log₂(7) ≈ 2.81 bits.

| Mode      | τ   | H (approx.)         |
|-----------|-----|---------------------|
| Relaxed   | 0.5 | ~1.0 bits/chord     |
| Challenge | 2.5 | ~2.7 bits/chord     |

### Corpus processing

The fitting pipeline (in `scripts/fit_transition_matrix.py`) applies three steps before counting any transitions:

1. **Chorus-only** — only chords from `<chorus_N>`-tagged sections are used. Choruses are purpose-built harmonic loops and give the cleanest signal.
2. **Per-instance boundaries** — each `<chorus_1>`, `<chorus_2>`, … is processed independently so repeated choruses don't create spurious cross-boundary transitions.
3. **Minimal repeating unit** — if a chorus contains the same 4-chord loop twice (e.g. C–G–Am–F–C–G–Am–F), only one copy is counted, and the loop length is recorded as 4, not 8.

The resulting length distribution is saved alongside the matrix and used at runtime to sample how many chords to generate (truncated at 6 to keep progressions tight).

## Usage

```bash
stochastic-strategies
```

Or directly via Python:

```bash
python -m stochastic_strategies.main
```

### Step-by-step

1. **Choose a difficulty**

   When prompted, type `Challenge` or `Relaxed`:

   - **Relaxed** — low entropy (τ = 0.5); progressions stay close to familiar voice-leading, diatonic extensions, common time signatures.
   - **Challenge** — near-maximum entropy (τ = 2.5); more chromatic passing chords, unusual extensions, odd time signatures.

2. **Receive your Creative Dilemma**

   Your card is printed to the terminal and contains:
   - A chord progression (key + diatonic degrees sampled from the Markov chain)
   - An Oblique-like songwriting prompt
   - A time signature and tempo
   - The entropy rate **H** of the chain at the chosen temperature

3. **Save your card (optional)**

   Enter `Y` to save — you'll be prompted for a filename (defaults to `creative_card`). The card is written as a `.txt` file in your current directory.

4. **Generate another card**

   If you skip saving, you'll be asked whether to generate a new card. Enter `Y` to loop, `N` to exit.

## Fitting the transition matrix

The `data/` directory ships with a pre-fitted matrix so you can run the tool immediately. To refit on the full Chordonomicon dataset (requires ~1 GB download and a few minutes):

```bash
pip install datasets matplotlib
python scripts/fit_transition_matrix.py
```

Optional filters:

```bash
# Pop songs only
python scripts/fit_transition_matrix.py --genre pop

# Pop songs from the 2010s
python scripts/fit_transition_matrix.py --genre pop --decade 2010

# Quick test on a subset
python scripts/fit_transition_matrix.py --max-songs 5000
```

Output is written to `data/transition_matrix.npy` and `data/transition_matrix_lengths.npy`. A heatmap of the fitted matrix is saved to `data/transition_matrix_heatmap.png`.

The notebook `notebooks/explore_chordonomicon.ipynb` walks through the dataset in detail — genre and decade distributions, key inference, per-decade entropy trends, and the most harmonically variable transitions across decades. A written summary of its findings is in [docs/notebook_insights.md](docs/notebook_insights.md).

## Tips

- Run it multiple times — the stochastic engine is designed to surprise you.
- **Relaxed** mode is great when you want a foundation to build on; **Challenge** mode works well as a forcing function when you feel stuck.
- Save cards that feel interesting even if you don't use them right away — they make good starting points later.
- The entropy score on the card tells you how "random" the progression is. A score near 2.8 bits means near-maximum harmonic unpredictability; a score near 1.0 means the chain is strongly pulling toward familiar resolutions.
